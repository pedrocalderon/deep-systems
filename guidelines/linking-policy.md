# Linking Policy

- Avoid internal path-based linking.
- Use tags for conceptual grouping.
- Only `catalog.md` may link directly to block overview files.
- Use Obsidian-style tags and backlinks instead of relative links.
- Use frontmatter fields id and aliases to link to a file, not full path.
