# AI Disclaimer

I use AI tools heavily in my day to day job, but since this is a public personal
project I will try to very clear what are the thinks that were AI generated and
what I hand crafted.

First of all, all the planning and the scaffolding of the project was done with
Chat GPT and Claude Code. I try to mark all the files as such, but I may have
missed some places.

## Indications

I will try to always use an indicator of AI generated content.

- `aigenerated: true`: in the frontmatter of markdown files
- comments at the top of any code file

For large parts of generated content I may have a centralized file. For exemple
for the local Neovim plugin, I added a single README.md at its root with the AI
generated disclaimer.

## Types of AI assistance

I will try to classify the AI usage of each file following the convention:

**VIBE CODED**: Fully AI generated, consider that I haven't even read that file.
**AI generated**: initially written by AI, I for sure read it and probably
edited. But it is manly an AI file that I'm working on top of.

I will not disclose other uses of AI. If I study a concept paring with Chat GPT
and hand craft a note, this note will not be considere AI generated. I did
absorb the knowledge and translated it to prose myself.

## General Guideline

Since the objective of this project is technical study it would not make a lot
of sense to use AI to write my code, so probably all the code you will see as
exercises will be handcrafted by me. This is not a judgement if that code could
have been written better or faster by the AI, it is just a study file in a
concept.

The same will apply to the text editor project, I will probably avoid code
assistants there.

I will however use LLMs freely to debug and understand concepts, it does not
make any sense to me to avoid this kind of help. In the end I will make use of
AI heavily if it helps my learning and will not use when it replaces the
abilities I'm trying to construct here.

I will use AI freely and without disclose for text review, improving my
vocabulary and presentation. Even generating charts of artifacts to help
understand what I have coded.

I will vibe code supporting features, like I did with the Neovim plugin added to
the project. This kind of ergonomic improvements is one of my favorite vibe
coding usages, I love to improve my workflow without loosing to much focus. I
would never have added a custom frontmatter formatter in this project if I
needed to build it myself. Since it is a very low stakes code I just accept the
output of the model and keep moving with my main objective.
